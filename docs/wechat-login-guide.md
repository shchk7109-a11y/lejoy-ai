# 微信一键登录接入指南

## 当前认证方案

乐享AI 当前使用 Manus OAuth 作为用户认证系统。该方案在 Web 端可以正常运行，支持用户注册、登录和会话管理。当应用部署为微信小程序时，需要额外接入微信原生登录能力。

## 微信小程序登录接入路径

在微信小程序环境中，推荐采用 **web-view 嵌套 H5** 的架构方案。小程序端负责微信授权登录，H5 端负责业务功能展示。具体实施步骤如下：

### 第一步：注册微信小程序

前往 [微信公众平台](https://mp.weixin.qq.com/) 注册小程序账号，获取 AppID 和 AppSecret。在小程序后台的"开发管理"中配置业务域名，将 H5 应用的域名添加到白名单。

### 第二步：小程序端实现登录

在小程序的 `app.js` 中调用 `wx.login()` 获取临时登录凭证（code），然后将 code 发送到后端服务器。后端使用 code 换取用户的 openid 和 session_key，完成用户身份识别。

### 第三步：后端对接

后端需要新增一个 `/api/wechat-login` 接口，接收小程序传来的 code，调用微信的 `jscode2session` 接口获取 openid，然后在数据库中查找或创建用户，签发 JWT Token 返回给前端。

### 第四步：H5 端适配

H5 页面通过 URL 参数或 postMessage 接收小程序传来的 Token，存储到 Cookie 或 localStorage 中，后续所有 API 请求携带该 Token 进行身份验证。

## 所需配置项

| 配置项 | 说明 | 获取方式 |
| :--- | :--- | :--- |
| 小程序 AppID | 小程序唯一标识 | 微信公众平台 → 开发管理 → 开发设置 |
| 小程序 AppSecret | 小程序密钥 | 同上，注意妥善保管 |
| 业务域名 | H5 应用的访问域名 | 需在小程序后台配置白名单 |

## 注意事项

微信小程序的 web-view 组件要求 H5 页面必须使用 HTTPS 协议，且域名必须在小程序后台的业务域名白名单中。此外，web-view 中的 H5 页面无法直接调用微信小程序的原生 API（如支付、分享等），需要通过 JSSDK 或 postMessage 进行通信。
